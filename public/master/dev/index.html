Logbook Plus API

Complete reference for all backend endpoints. Authentication uses server-side sessions with cookies. All protected routes require an active session from /api/login.

Base URLhttp://localhost:8080
Protocol HTTP / HTTPS
Auth Session Cookie
Body format JSON
File upload multipart/form-data
Quota 240 MB / user

🔑 Authentication

// No session required on these endpoints

POST /api/signup Create a new account ▼
Description
Registers a new user. The username field is stored as both the login ID and email. Returns plain text on success.
Request Body — application/json
Field Type Required Description
username string required Email or unique identifier used to log in.
password string required Plain text — bcrypt-hashed on the server.
name string optional Display name. Falls back to username if omitted.
Mobile Example
// Flutter + http package final res = await http.post( Uri.parse('http://localhost:8080/api/signup'), headers: {'Content-Type': 'application/json'}, body: jsonEncode({ 'username': 'user@example.com', 'password': 'secret123', 'name': 'Alex', }), ); // 201 → "Success! Account created."
const res = await fetch('/api/signup', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username, password, name }), credentials: 'include', });
Responses
201 Success! Account created.
400 User already exists.
500 Server Error
POST /api/login Start a session ▼
Description
Authenticates the user and sets a connect.sid session cookie. All subsequent protected calls must send this cookie.
📱 Mobile cookie tip: Flutter — use dio + cookie_jar + dio_cookie_manager. React Native — use @react-native-cookies/cookies with credentials: 'include'.
Request Body
Field Type Required Description
username string required Registered username/email.
password string required Plain text password.
Response Body (200)
{ "name": "Alex", "email": "user@example.com" }
Mobile Example
// Dio + CookieManager — cookie saved automatically final cookieJar = CookieJar(); final dio = Dio(); dio.interceptors.add(CookieManager(cookieJar)); final res = await dio.post( 'http://localhost:8080/api/login', data: {'username': email, 'password': pass}, ); final name = res.data['name'];
const res = await fetch('http://localhost:8080/api/login', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username, password }), credentials: 'include', });
Responses
200 { name, email }
401 Invalid Login
500 Server error
POST /api/logout Destroy session ▼
Description
Destroys the server-side session and clears connect.sid. No request body required.
Responses
200 Logged out
500 Logout failed

POST /api/backup 🔒 auth Upload a backup file ▼
Description
Upload as multipart/form-data. Saved as {policy}_{timestamp}.zip. Oldest file auto-removed if quota exceeded.
Form Fields
Field Type Required Description
file File required Any file — stored as .zip.
policy string optional Label prefix e.g. "Daily", "Weekly". Default: "Daily".
Mobile Example
final formData = FormData.fromMap({ 'file': await MultipartFile.fromFile(filePath, filename: 'backup.zip'), 'policy': 'Daily', }); final res = await dio.post('http://localhost:8080/api/backup', data: formData);
const fd = new FormData(); fd.append('file', { uri: fileUri, name: 'backup.zip', type: 'application/zip' }); fd.append('policy', 'Daily'); await fetch('/api/backup', { method: 'POST', body: fd, credentials: 'include' });
Responses
200 { message: "Backup saved" }
401 Unauthorized

DELETE /api/backup/:filename 🔒 auth Delete a specific backup ▼
Description
Permanently removes a file. Use the exact name from GET /api/backups. Path traversal rejected.
ℹ️ This route is implemented in server.js and is active by default.
Add to server.js
app.delete('/api/backup/:filename', isAuthenticated, async (req, res) => { const userDir = path.join(__dirname, 'uploads', req.session.userId); const filePath = path.join(userDir, req.params.filename); if (!filePath.startsWith(userDir)) return res.status(400).send("Invalid filename"); try { await fs.remove(filePath); res.json({ message: "Deleted" }); } catch (e) { res.status(500).send("Delete failed"); } });
Responses
200 { message: "Deleted" }
400 Invalid filename
401 Unauthorized
500 Delete failed

👑 Master Control

// Requires master session • add snippets below to server.js

Credentials live in env vars (MASTER_USER / MASTER_PASS). Login sets req.session.isMaster = true. All master routes are guarded by isMasterAuth middleware. This portal's lock screen validates against those same values.

Add once to server.js:
const MASTER_USER = process.env.MASTER_USER || 'admin'; const MASTER_PASS = process.env.MASTER_PASS || 'changeme123'; const isMasterAuth = (req, res, next) => { if (req.session.isMaster) return next(); res.status(403).json({ error: 'Master access required' }); };
POST /api/master/login 👑 master Authenticate as master ▼
Description
Sets req.session.isMaster = true. This is what the portal lock screen calls.
Request Body
Field Type Description
username string Value of MASTER_USER.
password string Value of MASTER_PASS.
Add to server.js
app.post('/api/master/login', (req, res) => { const { username, password } = req.body; if (username === MASTER_USER && password === MASTER_PASS) { req.session.isMaster = true; return res.json({ success: true }); } res.status(403).json({ error: 'Invalid master credentials' }); });
Responses
200 { success: true }
403 Invalid credentials
GET /api/master/users 👑 master List all users ▼
Description
All registered users with metadata. Passwords never returned. Includes per-user backup count and storage usage.
Response Body (200)
[{ "_id": "abc", "username": "user@e.com", "name": "Alex", "profilePicIndex": 2, "backupCount": 7, "storageMB": "142.35" }]
Add to server.js
app.get('/api/master/users', isMasterAuth, async (req, res) => { const users = await db.find({}, { password: 0 }); const result = await Promise.all(users.map(async u => { const dir = path.join(__dirname, 'uploads', u._id); let count = 0, size = 0; try { const files = await fs.readdir(dir); count = files.length; for (const f of files) size += (await fs.stat(path.join(dir,f))).size; } catch {} return { ...u, backupCount: count, storageMB: (size/1024/1024).toFixed(2) }; })); res.json(result); });
Responses
200 Array of user objects
403 Master access required
DELETE /api/master/user/:id 👑 master Delete user + all backups ▼
Description
Permanently deletes user record and entire backup directory. Irreversible.
Add to server.js
app.delete('/api/master/user/:id', isMasterAuth, async (req, res) => { await db.remove({ _id: req.params.id }, {}); await fs.remove(path.join(__dirname, 'uploads', req.params.id)); res.json({ message: 'User deleted' }); });
Responses
200 { message: "User deleted" }

GET /api/master/stats 👑 master Server-wide statistics ▼
Description
Aggregate totals: user count, backup count, total storage across all users.
Response Body (200)
{ "totalUsers": 42, "totalBackups": 318, "totalStorageGB": "7.24" }
Add to server.js
app.get('/api/master/stats', isMasterAuth, async (req, res) => { const users = await db.find({}, { _id: 1 }); let totalBackups = 0, totalSize = 0; for (const u of users) { try { const dir = path.join(__dirname, 'uploads', u._id); const files = await fs.readdir(dir); totalBackups += files.length; for (const f of files) totalSize += (await fs.stat(path.join(dir,f))).size; } catch {} } res.json({ totalUsers: users.length, totalBackups, totalStorageGB: (totalSize/1024/1024/1024).toFixed(2) }); });
Responses
200 { totalUsers, totalBackups, totalStorageGB }
403 Master access required
