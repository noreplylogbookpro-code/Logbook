Logbook Plus Self-Hosting

Complete deployment specification & integration setup guide

Deployment Quickstart

Logbook Plus self-hosted backend is written in lightweight Node.js. Deploy it locally on your Raspberry Pi, private server, or home lab.

  1. Clone or Copy the Server Repository to your private server.
  2. Install dependencies by running command:
    npm install
  3. Configure Environment Variables by editing or creating a .env file in the project root folder.
  4. Launch the Server using a production daemon manager like PM2:
    npx pm2 start server.js --name "logbook-server"

Environment Variables Configuration

Define these parameters in your .env file to enable license key checks and offline sync:

Variable Required Status Purpose / Value
PORT Optional Default Port (e.g. 8080)
IS_SELF_HOSTED Required Must set to true to enforce local license checks.
LICENSE_KEY Required Copy the JWT license key generated during your app purchase.
SESSION_SECRET Required Cryptographic session key. Change to a random long string.
JWT_SECRET Required User authorization token encryption key.

API Contract Specifications

To communicate properly with the Android application client, your self-hosted server exposes the following API routes:

Authentication Endpoints

  • POST /api/signup: Register a new local account. Input: { username, password, name, securityQuestion, securityAnswer }.
  • POST /api/login: Log in to fetch local JWT. Input: { username, password }. Returns standard JSON session token.

Sync & Cloud Vault Endpoints

  • POST /api/backup: Upload a local backup package (multi-part form data ZIP payload).
  • GET /api/backups: Retrieve all stored backups.
  • GET /api/restore/:filename: Download a selected backup ZIP for device restoration.
  • DELETE /api/backup/:filename: Permanently delete a backup from server storage.
